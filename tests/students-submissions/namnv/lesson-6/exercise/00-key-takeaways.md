## Tổng hợp kiến thức buổi 6

- **Quy trình tạo branch**
  - Luôn `git checkout main` và `git pull origin main` trước khi tạo branch mới.
  - Đặt tên branch theo convention của lớp, ví dụ: `feat/lesson-6-key-takeaways-namnv`, `feat/lesson-6-exercises-js-namnv`.

- **Tạo pull request (PR)**
  - Sau khi hoàn thành bài, commit và push code lên branch vừa tạo.
  - Trên GitHub, tạo PR từ branch đó vào `main`.
  - Điền **Title** ngắn gọn, **Description** mô tả chi tiết thay đổi.
  - Thêm **Reviewers** (mentor/bạn học) và **Assignees** (chính mình).

- **Review code**
  - Vào tab **Files changed**, đọc kĩ và comment vào những dòng có vấn đề.
  - Comment nên cụ thể: nêu rõ vấn đề, giải thích lý do và gợi ý hướng sửa.

- **Fix comment & hoàn tất PR**
  - Sửa code theo các comment hợp lý, commit & push thêm lên cùng branch.
  - Quay lại PR, bấm **Resolve conversation** cho những comment đã xử lý.
  - Khi reviewer thấy ổn sẽ Approve và merge PR vào `main`.
